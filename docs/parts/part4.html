Authentication with Azure App Services

Duration

5 minutes

Goals

In this demonstration, we will look at both the server-side implementation which adds filters to the queries issued by the client, as well as some minor changes to the client to hit the right endpoint.

Assets

There is a completed version of the client in the Exercise 4 folder. In addition, the Server folder included with these materials has a complete copy of the service we are calling.

Changes

You can explore the completed project on your own, here are the relevant changes made to the application to support refresh tokens - compare it to the prior exercise's Completed solution to get a full delta.

  1. The table endpoint was changed from tables/diaryentry2 to tables/diaryentry3 to get the support from the live system for query filtering on the server side.
  2. We've added a new UserId field to the DTO in DiaryEntry.cs.
  3. We need to call PurgeAsync at least once to clear the offline cache since we are now returning fewer records.
